## Mobile SEO Checklist

### 1. Responsive Design (Required)

**What is it?**
- Site adapts to screen size automatically
- Same URL for desktop and mobile
- Best practice (Google recommends)

**Check:**
- Open your site on phone
- Does it fit screen width (no horizontal scroll)?
- Is text readable (no zooming needed)?
- Are buttons tap-friendly (not tiny)?

**If NO:** Your site needs responsive design.

### 2. Page Speed (Critical)

**Mobile speed benchmark:**
- Good: Load under 4 seconds
- Excellent: Load under 3 seconds
- Poor: Load over 4 seconds

**Test:**
- PageSpeed Insights (pagespeed.web.dev)
- Test MOBILE speed specifically

**Common mobile speed issues:**
- Large images
- Too much code
- Slow hosting
- Too many plugins

**Fix:**
- Compress images
- Enable caching
- Minify code
- Upgrade hosting

### 3. Readable Text (No Zooming)

**Font size minimum:**
- Body text: 16px
- Headings: 24px+

**Check:**
- Can you read without zooming?
- Is text large enough on small screen?

**If NO:** Increase font size.

### 4. Tap-Friendly Buttons

**Button size minimum:**
- 48x48 pixels (Apple guideline)
- Easy to tap with thumb

**Check:**
- Can you tap buttons easily?
- Are buttons spaced apart (not too close)?

**If NO:** Enlarge buttons, add spacing.

### 5. No Flash or Pop-Ups

**Avoid:**
- Flash (doesn't work on mobile)
- Intrusive pop-ups (blocks content)
- Interstitials (full-screen ads)

**Google penalizes sites with intrusive pop-ups.**

**Instead:**
- Use HTML5 (not Flash)
- Non-intrusive CTAs (banner at bottom/top)
- Slide-in from side (doesn't block content)

### 6. Simplified Navigation

**Mobile menu:**
- Hamburger menu (3 lines icon)
- Collapsible sections
- Back buttons (easy navigation)

**Avoid:**
- Complex mega-menus (hard on mobile)
- Too many menu items (overwhelming)
- Deep navigation (hard to find content)

## Common Mobile SEO Issues

### Issue 1: Unplayable Content

**Problem:**
- Videos don't play
- Content requires Flash
- Interactive elements don't work

**Fix:**
- Use HTML5 video (not Flash)
- Ensure all media plays on mobile
- Test on multiple devices

### Issue 2: Faulty Redirects

**Problem:**
- Desktop page redirects to mobile homepage
- User loses content context

**Example:**
- Desktop: `domain.com/crane-rental`
- Mobile: `domain.com` (homepage - WRONG)

**Fix:**
- Use responsive design (same URL)
- Or redirect to equivalent mobile page

### Issue 3: Mobile-Only 404s

**Problem:**
- Page exists on desktop
- Shows 404 error on mobile

**Fix:**
- Ensure all pages accessible on mobile
- Test mobile crawlability

### Issue 4: Blocked CSS/JavaScript

**Problem:**
- robots.txt blocks CSS/JS files
- Google can't render page properly

**Fix:**
- Allow Googlebot to access CSS/JS
- Don't block resources

### Issue 5: Slow Mobile Speed

**Problem:**
- Desktop loads fast
- Mobile loads slow

**Causes:**
- Heavy images
- Too many scripts
- Slow mobile networks

**Fix:**
- Optimize images (WebP format, compression)
- Lazy loading (load images as user scrolls)
- Reduce script size
