## ğŸ” Constraints & Limitations

### Current Constraints

**Commands:**
- âŒ No complex logic (if/else branches)
- âŒ Limited parameters (max 2-3)
- âŒ Single agent per command
- âŒ No validation in command layer

**Agents:**
- âŒ No memory between runs (stateless)
- âŒ Sequential execution only (not parallel)
- âŒ No error retry logic
- âŒ No cross-client learning
- âŒ File system only (no database)

**Skills:**
- âŒ Static files (no dynamic updates)
- âŒ Text instructions only (no code)
- âŒ Manual maintenance required
- âŒ No A/B testing built-in
- âŒ Agent interpretation dependent

### Why These Constraints Exist

**Simplicity > Features:**
- Easy to understand for non-technical users
- Easy to debug (just read markdown files)
- Easy to version control (Git)
- Easy to customize (edit text files)

**Trade-off:** Less sophisticated, but 10x easier to maintain
