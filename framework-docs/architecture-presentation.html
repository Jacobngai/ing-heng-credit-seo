daily-seo-content.md

Contains: Master workflow for blog generation

  • Read client brief.json and personas
  • Use keyword-research.md skill for SERP analysis
  • Use multi-language-content.md for each language
  • Apply content-quality-check.md standards
  • Save drafts with D/ prefix

keyword-research.md

Contains: How to research keywords per language

  • Search Google for target keyword
  • Analyze top 10 SERP results
  • Extract LSI keywords and topics
  • Identify content gaps
  • Language-specific: EN vs ZH vs MS approach

multi-language-content.md

Contains: How to culturally adapt content

  • NOT translation - native content generation
  • Cultural references and idioms per language
  • Local SEO optimization (Google vs Baidu)
  • Examples: EN (ROI-focused), ZH (relationship), MS (respect)
  • CTA adaptation per culture

content-quality-check.md

Contains: Quality standards checklist

  • Word count: 1500-2500 words
  • Readability: Flesch 60+ (Grade 8-9)
  • Keyword density: 1-2% for primary keyword
  • Structure: H1 + 4-6 H2s + H3s
  • Grammar and spelling check

publishing-workflow.md

Contains: How to deploy approved content with multi-website support

  • Find all files with A/ prefix (approved)
  • Load deployment config from status.json for each client
  • Route content by language to correct website (multi-website support)
  • Generate final blog posts with metadata
  • Move to /website-[identifier]/src/content/posts/
  • Git add, commit with message template (per website)
  • Push to GitHub main branch (triggers auto-deploy)
  • Monitor deployments using Vercel MCP
  • Verify deployment status (READY/BUILDING/ERROR)
  • Check build logs if errors occur
  • Confirm URLs accessible (HTTP 200)

client-onboarding.md

keyword-research.md

Contains: Industry-specific keyword research methodology

  • Analyze competitors (top 5 in client's industry)
  • Research 100+ keywords per language (EN/ZH/MS)
  • Map keywords to awareness stages (awareness/consideration/decision)
  • Identify long-tail opportunities
  • Calculate search volume and difficulty
  • Output: D/primary-keywords.md, D/long-tail-keywords.md, D/competitor-analysis.md

persona-generation.md

Contains: How to create detailed buyer personas

  • Based on target audience from questionnaire
  • Generate 5-8 persona profiles with: demographics, psychographics, pain points, goals, objections
  • Include content preferences per persona
  • Map personas to buyer journey stages
  • Language preferences per persona
  • Output: D/contractor-bob.md, D/logistics-lee.md, D/warehouse-william.md, etc.

brief-template.md

Contains: Client brief structure and content

  • Company overview and positioning
  • SEO strategy and goals
  • Content calendar suggestions (frequency, topics)
  • Language distribution (e.g., 40% EN, 30% ZH, 30% MS)
  • Publishing schedule and quotas
  • Brand voice guidelines
  • Output: D/brief.md

PHASE 1.5 SKILLS - Image Analysis & Brand Identity

image-analysis.md

Contains: OCR and AI vision analysis for client photos

  • Analyze uploaded photos using AI vision (Claude multimodal)
  • Extract visual content (equipment type, brand, model, condition, setting)
  • Run OCR text extraction (model numbers, specifications, visible text)
  • Extract keywords: primary, secondary, long-tail (language-specific)
  • Identify content gap opportunities (blog ideas, landing pages, FAQs)
  • Generate SEO metadata (alt text, title, file renaming suggestions)
  • Create comprehensive image brief per photo
  • Output: D/image-briefs/*.md (one per image)

image-keyword-integration.md ‚≠ê CRITICAL

Contains: Merge image keywords with existing keyword research

  • Why this matters: Without this, image keywords are ORPHANED (documented but not used!)
  • Extract all keywords from approved image briefs
  • Load existing keyword research from Phase 1 (A/ files)
  • Compare and merge: mark duplicates as "validated by visual assets"
  • Add new keywords discovered from images
  • Re-prioritize combined keywords (visual assets get higher priority)
  • Generate enhanced keyword research files with 30-50% more keywords
  • Identify content opportunities enabled by having real images
  • Output: D/keyword-research/enhanced-*.md (5 files for review)

üåê Multi-Language Deployment Architecture

üéØ Problem Statement

Client Challenge: Need separate Vercel deployments per language (e.g., English-primary, Chinese-primary, Malay-primary) but want to avoid duplicating content and code.

Solution: One codebase, multiple builds - like environment configs (dev/staging/prod), but for languages (en/zh/ms).

üö´ No Duplication

Content written once in /content/[lang]/, used by all deployments. Single source of truth.

‚ö° Parallel Deployment

Push once ‚Üí all language sites update simultaneously. 5 min total (not 15 min).

üåç Language-Specific Domains

Each language has its own primary domain: apexmachinery.com, zh.apexmachinery.com, ms.apexmachinery.com

üìà Better SEO

Each language is "primary" on its own deployment. Better local targeting and ranking.

How It Works: Build-Time Configuration

Each Vercel Project Has:

  • Different Git Branch: deploy-en, deploy-zh, deploy-ms
  • Different Build Command: npm run build:en, npm run build:zh, npm run build:ms
  • Different Default Language: Set via BUILD_LANG env var
  • Same Codebase: All branches mirror main (no code duplication)

Build Process:

Vercel Project 1 (English-primary):
Git branch: deploy-en
Build: BUILD_LANG=en astro build
Output: apexmachinery.com/     ‚Üê English content (no /en/ prefix)
        apexmachinery.com/zh/  ‚Üê Chinese content
        apexmachinery.com/ms/  ‚Üê Malay content

Vercel Project 2 (Chinese-primary):
Git branch: deploy-zh
Build: BUILD_LANG=zh astro build
Output: zh.apexmachinery.com/     ‚Üê Chinese content (no /zh/ prefix)
        zh.apexmachinery.com/en/  ‚Üê English content
        zh.apexmachinery.com/ms/  ‚Üê Malay content

Vercel Project 3 (Malay-primary):
Git branch: deploy-ms
Build: BUILD_LANG=ms astro build
Output: ms.apexmachinery.com/     ‚Üê Malay content (no /ms/ prefix)
        ms.apexmachinery.com/en/  ‚Üê English content
        ms.apexmachinery.com/zh/  ‚Üê Chinese content

Publishing Workflow Update

When /publish-approved command runs:

1. Find all approved content (A/ files)
2. Copy to /content/[lang]/blogs/
3. Git commit to main branch
4. Push to ALL deployment branches:
   git push origin main:deploy-en --force
   git push origin main:deploy-zh --force
   git push origin main:deploy-ms --force
5. Vercel auto-deploys ALL 3 in parallel (5 min total)

Result:
‚úÖ apexmachinery.com updates
‚úÖ zh.apexmachinery.com updates
‚úÖ ms.apexmachinery.com updates

Content written ONCE, deployed THREE times!

Adding New Languages (Scalable)

To add a 4th language (e.g., Tamil):

  1. Create config/deploy-ta.json
  2. Add "build:ta": "BUILD_LANG=ta astro build" to package.json
  3. Create Git branch: git checkout -b deploy-ta
  4. Create Vercel project pointing to deploy-ta branch
  5. Create content folder: content/ta/

Time to add new language: ~10 minutes. No code changes needed!

  • Maintainability: Fix bugs once, applies to all deployments
  • Consistency: All language versions use same codebase
  • Performance: Parallel deployment (5 min for all vs 15 min sequential)
  • Scalability: Add new languages in 10 minutes
  • SEO: Each language is "primary" on its own domain
  • Cost: Multiple Vercel free tier projects (shared build cache)
